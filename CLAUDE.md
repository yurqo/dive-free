## Project overview

DiveFree is a SwiftUI freediving session logger for Apple Watch Ultra + iPhone. It records depth via `CMWaterSubmersionManager`, detects dive events, saves sessions via SwiftData, and syncs to the iPhone via WatchConnectivity. It is not a dive computer.

Swift 6, strict concurrency (`SWIFT_STRICT_CONCURRENCY = complete`), `@Observable` + `@MainActor` throughout.

## Workspace setup

The `.xcodeproj` and `.xcworkspace` are **not committed** — generated by [Tuist](https://tuist.dev) from `Project.swift`. Always regenerate after changing `Project.swift` or after a fresh clone:

```sh
tuist generate --no-open   # regenerate workspace
open DiveFree.xcworkspace
```

Tuist version is pinned in `mise.toml`. CI installs it via `mise`.

## Commands

```sh
# Run all package tests (iOS simulator)
tuist test

# Run tests for a single package
xcodebuild test -workspace DiveFree.xcworkspace -scheme Domain \
  -destination "platform=iOS Simulator,name=iPhone 17" CODE_SIGNING_ALLOWED=NO | xcbeautify

# Build the Watch app (do this after any #if os(watchOS) change — see below)
xcodebuild build -workspace DiveFree.xcworkspace -scheme DiveFreeWatch \
  -destination "generic/platform=watchOS Simulator" \
  CODE_SIGNING_ALLOWED=NO EXCLUDED_ARCHS=x86_64 | xcbeautify

# Build the iPhone app
xcodebuild build -workspace DiveFree.xcworkspace -scheme DiveFree \
  -destination "generic/platform=iOS Simulator" \
  CODE_SIGNING_ALLOWED=NO EXCLUDED_ARCHS=x86_64 | xcbeautify
```

`EXCLUDED_ARCHS=x86_64` is required on Apple Silicon — the runner produces arm64-only slices.

## Architecture

### Module graph

```
DiveFree (iPhone)  DiveFreeWatch (Watch)
        \                  /
    Strava  Persistence  Sensors  Sync
                  \       |      /
                        Domain
```

`Domain` is dependency-free: pure Swift models (`DepthSample`, `DiveSession`, `Dive`, etc.) and `DiveDetector`. Everything else can depend on `Domain`; packages do not depend on each other.

### Key types

| Type | Package | Role |
|---|---|---|
| `DiveDetector` | Domain | Detects dive events from a `[DepthSample]` array |
| `SensorManager` | Sensors | `@Observable` façade; owns a `DepthProvider`, publishes `currentDepthMeters` |
| `WaterSubmersionDepthProvider` | Sensors | watchOS-only; bridges `CMWaterSubmersionManager` delegate into `AsyncStream<DepthSample>` |
| `MockDepthProvider` | Sensors | Synthetic depth profile; used on iOS simulator and in tests |
| `DiveStore` | Persistence | SwiftData store for `DiveSession` |
| `SyncManager` | Sync | WatchConnectivity; sends `DiveSession` from Watch → iPhone |
| `SessionCoordinator` | WatchApp | Glue layer: starts `WorkoutController` + `SensorManager`, runs `DiveDetector`, calls `SyncManager` |
| `WorkoutController` | WatchApp | `HKWorkoutSession` + `HKLiveWorkoutBuilder` lifecycle |

### Dependency injection

`makeDepthProvider()` (in `Sensors`) returns `WaterSubmersionDepthProvider` when `CMWaterSubmersionManager.waterSubmersionAvailable` is true, otherwise `MockDepthProvider`. `SensorManager` accepts any `DepthProvider`, making it testable.

### Test targets

All test targets build for **iOS only** (`Project.swift` sets `destinations: .iOS`). This means `#if os(watchOS)` code is never compiled in tests. Extract platform-agnostic logic (e.g. pure transform functions) outside the `#if` guard to keep it testable.

## Important build rule

**After editing any `#if os(watchOS)` code, build `DiveFreeWatch` locally before pushing.** The iOS test schemes never compile watchOS-gated code, so errors only surface on CI otherwise.
